## Design Context

### Users
Notion 重度用户，主要场景是**边浏览边存、随手收藏**。他们在快速滚动信息流时发现有价值的内容，希望以最少的操作将其归档到 Notion。对速度敏感，不希望被复杂 UI 打断浏览节奏。次要场景是批量整理多个链接。

### Brand Personality
**高效 · 轻巧 · 克制**

- 高效：一点即存，不拖慢工作流
- 轻巧：小蚂蚁隐喻——默默工作、不喧哗的自动化工具
- 克制：UI 只展示用户当前需要的，不抢焦点

情感目标：用户每次存完书签，感觉"好，搞定了"——满足感而非惊喜感。

### Aesthetic Direction
**极简绿色系，功能优先。**

- 主色：`#87BD09`（自然绿，暗示生产力与自动化），hover 加深为 `#76A903`
- 背景：`#f9f9f9`，输入框白色——干净、无干扰
- 字体：系统字体栈，无自定义字体（保持轻量、跨平台一致）
- 只有亮色主题，暂不支持 Dark Mode
- 蚂蚁图标（🐜）作为唯一品牌符号，保持存在感但不张扬
- **反参考**：不要 Notion 官方的白色极简风（太平淡）；不要夸张的渐变/阴影（太重）；不要任何红色/警报感视觉

### Design Principles

1. **零打断原则**：任何 UI 变化（状态提示、错误信息）都不应让用户觉得"我需要处理这个"，用最轻量的方式传达信息。

2. **绿色即行动**：`#87BD09` 只用于"可以点击/已激活/成功"的状态，不做装饰用途。视觉引导用户的注意力直指行动点。

3. **渐进披露**：默认状态保持最小 UI，高级功能（批量模式、封面图、推文模式）通过 toggle 按需展开，避免新用户认知负担。

4. **克制的反馈**：成功/失败消息简短、不打断操作流；不弹窗、不模态对话框。

5. **输入框即主角**：所有视觉权重导向输入区域和主按钮，其余元素退居背景。

## Tech Stack
- Chrome Extension, Manifest V3
- 纯 HTML5 + CSS3 + Vanilla JavaScript（无框架、无 npm 依赖）
- Popup 固定宽度 350px

## Current Color Tokens
| Token | Value | Usage |
|-------|-------|-------|
| Primary | `#87BD09` | 主按钮、toggle 开启、焦点边框 |
| Primary Hover | `#76A903` | 主按钮 hover |
| Background | `#f9f9f9` | Popup 背景 |
| Surface | `#ffffff` | 输入框背景 |
| Border | `#dddddd` | 输入框边框 |
| Warning | `#f5a623` | 无封面图提示 |
| Text Primary | `#333333` | 标题、正文 |
| Text Secondary | `#666666` | 标签 |
| Text Hint | `#999999` | 占位文字 |
| Disabled | `#cccccc` | 禁用状态 |
